Selection boxes present users with choices or questions that they must answer before they proceed with one-time processes.

Use selection boxes when users must make a choice or answer a standalone question before proceeding with a one-time process such as a setup or configuration task. Selection boxes represent the only decision on a page.

Use selection boxes when options are abstract or potentially unfamiliar to users. Use icons and descriptions to provide additional insight.

Don’t use selection boxes for repeatable tasks such as data entry. Use other selection controls, such as checkboxes or radio buttons, instead.

Don't use selection boxes when there are more than 12 options. Use other selection controls instead.

Icon

Include icons when they add meaning that helps users distinguish the options. If icons don’t add useful meaning to the options, don’t include them.

When using icons, use one for every option in the group of selection boxes. Don’t reuse the same icon for more than one option in a group.

Include descriptions when they add meaning that helps users distinguish the options.

When using descriptions, use one for every option in the group of selection boxes.

Don't use selection boxes with no icon or description. If neither the icon nor the description add meaning, use checkboxes or radio buttons instead.

On smaller viewports, selection boxes hide their icons and use smaller typography and spacing.

Keep titles and descriptions short and concise. Limit descriptions to a single sentence.

Selection boxes are arranged in a centered container based on their number and the container’s width. Don’t use different layouts with selection boxes.
